# What I Learned: Archgate Init and ADR Migration

**Date:** 2026-03-27

## What I Learned This Session

### Archgate stores ADRs under `.archgate/adrs/`, not `docs/adr/`

After running `archgate init`, the canonical ADR location becomes `.archgate/adrs/`. The old `docs/adr/` directory (e.g., `ADR-001-webhook-driven-agent-architecture.md`) was deleted from the repo because archgate takes over ADR ownership. This is a **breaking change** to existing ADR file paths — any tooling or references pointing to `docs/adr/` will break.

### Archgate ADR frontmatter format is different from plain ADRs

Plain markdown ADRs in this project had no frontmatter. Archgate ADRs require specific frontmatter:

```markdown
---
id: GEN-001
title: Example Architecture Decision
domain: general
rules: false
---
```

- `id`: Prefix is the domain abbreviation (e.g., `GEN` for general), followed by a zero-padded number.
- `domain`: Categorizes the ADR (e.g., `general`, `api`, `frontend`).
- `rules`: Set to `true` to enable automated compliance checking via a companion `.rules.ts` file.

### ADRs can enforce themselves via `.rules.ts` files

When `rules: true` is set in an ADR's frontmatter, archgate expects a companion `.rules.ts` file alongside the ADR. The rule is written against the `RuleContext` interface (defined in `.archgate/rules.d.ts`, which is auto-generated — do not edit). The rule can use `ctx.glob()`, `ctx.grep()`, `ctx.readFile()`, and `ctx.report` to check violations.

### `.archgate/rules.d.ts` is auto-generated — never edit it

This file declares TypeScript interfaces (`RuleContext`, `RuleConfig`, `RuleSet`, etc.) used by `.rules.ts` files. It is regenerated by archgate commands and any manual edits will be lost.

### Archgate has a version check on init

Running `archgate init` reports if an upgrade is available (e.g., `0.26.1 → 0.26.2`). The upgrade command is `archgate upgrade`.

## What a New Team Member Should Know

- After `archgate init`, **do not create ADRs in `docs/adr/`** — put them in `.archgate/adrs/` using the frontmatter format above.
- To add automated rule enforcement to an ADR: set `rules: true` in frontmatter and create a sibling `.rules.ts` file implementing the `RuleSet` interface.
- The `rules.d.ts` types file is generated — import types implicitly via the ambient declarations, don't import from the file directly.
- The `.archgate/lint/` directory is where custom lint rules live (separate from ADR-coupled rules).

## Docs & Info That Would Speed Things Up Next Time

- Archgate ADR format spec: referenced in the example ADR at `.archgate/adrs/GEN-001-example.md` → `https://archgate.dev/docs/adr-format`
- Type signatures for writing `.rules.ts` files: `.archgate/rules.d.ts`
- Existing example rule to copy from: `.archgate/adrs/GEN-001-example.md` (shows the template structure)
