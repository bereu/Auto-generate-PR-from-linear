# What I Learned: docs/adr and .archgate/adrs Symlink Conflict

**Date:** 2026-03-27

## What I Learned This Session

### CLAUDE.md describes a symlink that doesn't actually exist yet

`CLAUDE.md` states `docs/adr/` is "symlinked from `.archgate/adrs/`", but after `archgate init`, **both are real directories** — not a symlink relationship. The symlink is aspirational documentation, not a fact.

You cannot `ln -s` over an existing directory. Attempting to create the symlink will fail silently or error unless you explicitly remove one of the directories first.

### GEN-001-example.md in .archgate/adrs/ is auto-generated by archgate init

When `archgate init` runs, it places a `GEN-001-example.md` file in `.archgate/adrs/` as a starter template. This file is **not user content** — it is archgate scaffolding. Do not confuse it with real ADRs.

### The symlink setup requires an explicit, destructive step

To make `docs/adr/` a symlink to `.archgate/adrs/`, you must choose a direction and remove one directory:

**Option A — symlink `docs/adr` → `.archgate/adrs`:**

```bash
rm -rf docs/adr
ln -s ../.archgate/adrs docs/adr
```

**Option B — symlink `.archgate/adrs` → `docs/adr`:**

```bash
# Move GEN-001-example.md to docs/adr first
mv .archgate/adrs/GEN-001-example.md docs/adr/
rm -rf .archgate/adrs
ln -s ../../docs/adr .archgate/adrs
```

This decision is irreversible in the sense that whichever directory you remove, the other becomes the canonical location.

### ADR-001 was deleted from docs/adr/ during archgate migration

`docs/adr/ADR-001-webhook-driven-agent-architecture.md` was deleted as part of migrating to archgate. If it needs to be recovered, check `git log -- docs/adr/ADR-001-webhook-driven-agent-architecture.md`.

## What a New Team Member Should Know

- Do not trust CLAUDE.md's claim that `docs/adr/` is already symlinked to `.archgate/adrs/` — verify with `ls -la docs/adr` before assuming.
- `archgate init` creates `.archgate/adrs/` as a real directory with a sample file. The symlink to `docs/adr/` is a **separate manual step**.
- Before creating the symlink, decide which directory is canonical and migrate any content from the one you'll remove.
